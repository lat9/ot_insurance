Prerequisites

This version of the Optional/Required Shipping Insurance requires:

  1. A working copy of Zen Cart 2.1.0 or later.
  2. The site must use a PHP version greater than or equal to PHP 8.0.
  3. (optional) A working copy of Edit Orders, v5.0.0 or later.

Overview

This Zen Cart order-total module enables a site to include an optional (or required) shipping-insurance fee for a customer's order.

Installation

Follow these steps to install the order-total:

  1. Unzip the plugin's zip-file distribution.
  2. Copy the contents for the distribution's includes directory to your site's /includes directory.
  3. Log into your Zen Cart admin and define your shipping zone definitions prior to installing this module.
  4. Navigate to the admin's Modules :: Order Total; you'll see an order-total named Shipping Insurance. Click on the associated Install button, and configure the order-total.
Uninstalling

To uninstall the order-total:

  1. Log into your Zen Cart admin and navigate to Modules :: Order Total.
  2. Locate the entry for Shipping Insurance, click the associated Un-Install button, confirm the removal and you've removed the order-total's processing.
  3. Remove the two files that you copied when installing the order-total.

Configuring

Note: If you need to add a zone after this module is installed, you will need to save your configuration settings to a text file and reinstall the module through the admin.

The following settings are available for the shipping-insurance calculations:

  1. Enable Insurance Module: Set this to true to enable the insurance module; default false.
  2. Sort Order: Sort order of display. Note: Must be higher than the sub-total.
  3. Use Table Rates? Set this to true (the default) if you want to define different rates for each of your defined zones. Set to false to use the increment or percentage methods.
  4. Alternate Insurance Type: If not using "Table Rates", set to percent (the default) to return insurance rate as a percentage of the order total. Set to amount to use the increment method below.
  5. Insurance Percentage: If using the percentage-based insurance type, enter the percentage of the order's total to be added for insurance.
  6. Insurance Rate: If not using the percentage-based insurance type, enter the rate to charge per every X amount (defined in the next section) of the order total.
  7. Increment Amount: Used by non-percentage-based insurance types. Identify the amount-increment for which the Insurance Rate is applied. If, for example, the increment-amount is 100 and the rate is .50, the insurance is calculated as $0.50 for each $100.00 of the total.
  8. Amount Exempt From Fee: Used with amount calculations. Set this to the amount of the total that is exempt from the insurance calculations, i.e. set to 100 for all orders under 100 to be exempt, already insured, etc.
  9. Tax Class: Use the selected tax-class for the insurance fee.
  10. No Insurance Fee on Virtual Products: Set to true (the default) if the price of any virtual product(s) in the order is not to be included in the insurance calculation.
  11. No Insurance Fee on Gift Vouchers: Set to true (the default) if the price of any gift voucher(s) in the order is not to be included in the insurance calculation.
  12. No Insurance Fee on Free Shipping: Set to true (the default) to not include the prices of items with free shipping in the insurance calculation. The insurance rate returned will be calculated on only products that return a shipping rate. This will make it so that all virtual and gift vouchers are not calculated as part of the order total.
  13. Required Insurance Amount: If the order total is above this amount, shipping insurance is required and is automatically calculated in the order total without the option to be able to remove it by the customer.
  14. Table Rates: A rate table is created for each Zone you have defined in your admin. Please follow the example given in the configuration page for instructions on how to use the table rates.
